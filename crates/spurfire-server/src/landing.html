Inside a lobby

A temporary posse, not a game server.

The control plane creates, credentials, inspects, reconciles, and deletes the network boundary. It does not join that network. Once connected, gameplay travels between riders instead of bouncing through Ottawa.

1

Create

A lobby record fixes capacity and expiry. Real isolation uses one provider tailnet dedicated to that lobby.

2

Join once

Each rider gets one short-lived, non-reusable enrollment credential. Organization credentials stay server-side.

3

Connect

RustScale gives every peer a private endpoint and chooses direct UDP, Peer Relay, or DERP Relay.

4

Elect

Fresh route, latency, loss, and device inputs produce a deterministic authority reference.

5

Ride & retire

Authority can migrate after host loss. Cleanup is complete only after exact network absence is verified.

Peers & freshness

Directional routes

Rows are directional. Peer Relay and DERP Relay are distinct, and no reverse path is inferred.

Application quality

RTT means application nonce/reply RTT only. Unknown measurements stay unknown, never zero.

Authority

Formula output, accepted heartbeat receipt, and peer-reported match authority are separate facts. Reports are not ranked proof.

Cleanup

Destroyed lobby state or a delete acknowledgement alone does not prove a dedicated tailnet is absent.

No packet capture is used. Provider observations are coarse metadata, participant reports remain authenticated-but-untrusted claims, and the control service is not a tailnet member, gameplay peer, relay, witness, or gameplay server.

Connection truth

Know how riders are reached.

A participant report can describe each directional gameplay route and application quality. It cannot turn a hostname, tailnet membership, or a device count into trusted player identity.

DIRECTEnd-to-end peer traffic without inferring that the reverse direction uses the same route.
PEER RELAYA RustScale peer-relayed path, labeled precisely rather than as generic relay.
DERP RELAYA provider relay path, distinct from Peer Relay and not a substitute for application RTT.
UNKNOWN / STALEA missing or aged observation remains explicit instead of becoming false offline or zero.
Lobby metadata

What the lobby knows—and what it does not.

Tailnet FQDNs and private addresses are topology metadata, not credentials, but they still require an authorized audience. This inspector omits private endpoints and operator-only provider identity.

Visible to an authorized lobby member

Bounded, freshness-labeled facts needed to understand one selected session.

  • Lobby and independent network lifecycle
  • Tailnet DNS name/FQDN while active or cleanup is pending
  • Qualified enrollment, reporter, and route counts
  • Application RTT/loss, authority references, and safe cleanup state

Never rendered by this inspector

Secrets, raw provider inventory, and unrelated topology stay outside the view.

  • No auth key, OAuth material, bearer token, or node private key
  • No raw provider device ID, tag, response, or stable tailnet ID
  • No physical/NAT endpoint, private application endpoint, or packet content
  • No other lobby, public directory, or operator reconciliation detail
